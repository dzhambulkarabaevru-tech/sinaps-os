## Запуск

```bash
npm install
cp .env.example .env.local   # заполнить ANTHROPIC_API_KEY, SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY
# выполнить lib/db/schema.sql в Supabase SQL editor
npm run dev
```

## Деплой на Netlify

1. Залить проект в GitHub-репозиторий (Netlify деплоит из Git)
2. **netlify.com** → «Add new site» → «Import an existing project» → выбрать репозиторий
3. Netlify определит `netlify.toml` и подключит `@netlify/plugin-nextjs` автоматически
4. Site settings → Environment variables — задать `ANTHROPIC_API_KEY`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`
5. **Обязательно платный план (Pro и выше)** — пайплайн M0→M10 делает несколько последовательных вызовов Claude подряд (20–40+ сек), бесплатный план Netlify (лимит 10 сек на функцию) будет обрывать запрос. В `netlify.toml` таймаут для `/api/create-path` уже поднят до 26 сек — это максимум, доступный на платном плане
6. Deploy — после первого запуска обязательно проверить: а) что `knowledge-base/` действительно попала в бандл функции (карта роста и библиотека должны влиять на генерацию, не быть пустыми), б) что реальный вызов «Создать путь» укладывается по времени и не обрывается 502-ошибкой

**Если запросы всё равно обрываются по таймауту** — самый надёжный вариант не Netlify-специфичный: снизить `MAX_QUALITY_ITERATIONS` в `lib/pipeline/orchestrator.ts` с 2 до 1 (один цикл M7↔M8 вместо двух). Это меняет поведение системы (более быстрый, но потенциально менее отточенный контент) — не стал делать это самостоятельно, так как это осознанный компромисс между скоростью и качеством, а не чисто техническая правка.

## Что менять, когда придут документы 4–8

- **Документ 4 (Карта роста подробная)** → `UPDATE growth_map_stages SET description = ...` — миграция структуры не нужна
- **Документ 5 (Контентный движок)** → только системный промпт в `lib/modules/m7-content-engine.ts`, сигнатура не меняется
- **Документ 6 (Библиотека)** → наполнение `knowledge-base/content-library/`, при росте — перенос в БД
- **Документ 7 (Контроль качества)** → системный промпт в `lib/modules/m8-quality-check.ts` + дополнение `knowledge-base/style-rules.md`
- **Документ 8 (UX-прототип)** → доработка компонентов в `components/`, логика пайплайна не меняется
